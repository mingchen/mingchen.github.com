使用windbg来调试程序

Apr 24th, 2007 | Comments

windbg是Windows上的调试工具。据说功能比较强大，我用它是因为它能支持JIT和 利用程序的mini dump来进行Postmortem调试。 发现其确实是一个不错的工具。

其它命令

JIT(Just In Time)调试

执行如下命令把windbg注册为默认的调试器：

windbg -I

它会修改如下的注册表项：

HKLM\Software\Microsoft\Windows NT\CurrentVersion\AeDebug

设置之后windbg就成为默认的调试器了，当程序挂了之后他就会跳出来。 如果你不想要它自动调用windbg，可以把注册表项下面的Auto这个值改为0， 这样就会先弹出一个对话框问你是否调试了。

分析mini-dump文件

可以使用windbg来分析mini-dump文件,使用-z参数即可。 比如要调试一个x.dmp的mini-dump文件，可以使用如下的命令:

windbg -z x.dmp -c "!analyze -v; q" -o logo x.txt

Debian 的 Apt-get 系列命令

Apr 17th, 2007 | Comments

配置文件

APT的主配置文件为/etc/apt/sources.list

更新数据库

apt-get update

如果出现下面的错误：

E: Dynamic MMap ran out of room

把下面这行加入到/etc/apt/apt.conf里面去就可以了。

APT::Cache-Limit "10000000";

搜索软件

apt-cache search pkgname

示例：

apt-cache search kernel-image

安装软件

apt-get install pkgname

示例：

apt-get install kernel-image-2.6-686
apt-get install python lynx wget ntp ntp-server ntpdate

升级软件

apt-get upgrade pkgname

示例:

apt-get upgrade glibc6

删除软件

apt-get remove pkgname

清理下载的安装文件

apt-get clean

这条命会释放硬盘空间，把这两个文件夹下的文件删除掉啦：

/var/cache/apt/archives/
/var/cache/apt/archives/partial/

最后的招数

/usr/sbin/base-config

附：一次升级过程(升级到2.6的内核)

apt-get dist-upgrade
apt-get install kernel-image-2.6-686
apt-get install ssh ftp vsftpd wget ntp ntp-server ntpdate vim \
    cdecl ctags gcc g++ gcc-4.1-base gdb make flex bison python \
    lynx  perl subversion cvs openssl manpages-posix manpages-posix-dev bzip2

如果遇到下面的错误：

E: This installation run will require temporarily removing the essential package e2fsprogs
due to a Conflicts/Pre-Depends loop. This is often bad, but if you really want to do it,

activate the APT::Force-LoopBreak option.
E: Internal Error, Could not early remove e2fsprogs

解决方法：

apt-get -o APT::Force-LoopBreak=yes install e2fsprogs
apt-get -f install # 一路回车就行了

然后再重新执行遇到错误的命令

升级内核过程中的重要提示：

configure LILO, you need to add an 'initrd=/initrd.img' to the
image=/vmlinuz stanza of your /etc/lilo.conf

I repeat, You need to configure your boot loader

NTP MINI HOWTO

Apr 14th, 2007 | Comments

我的计算机需要一个准确的时间, 所以我选择了NTP

安装ntp (Debian Linux)

# apt-get install ntp
# apt-get install ntp-server
# apt-get install ntpdate

配置

配置文件：/etc/ntp.conf

可用的NTP服务器：可以到http://www.pool.ntp.org上面去找。我用的如下

server 0.asia.pool.ntp.org
server 1.asia.pool.ntp.org
server 2.asia.pool.ntp.org
server 3.asia.pool.ntp.org
time.nist.gov

把ntpdate加入到root的cron里面去，让它每小时运行一次:

# crontab -e

加入下面的一行

0 * * * * ntpdate time.nist.gov pool.ntp.org cn.pool.ntp.org

后台进程名:

ntpd

实用程序ntpq

ntpq可以用来查询NTP服务器的状态

示例：

ntpq -q time.nist.gov
